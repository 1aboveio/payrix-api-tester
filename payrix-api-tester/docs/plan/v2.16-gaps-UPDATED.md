## Critical Gaps (Must Fix)

### 1. 🔴 Tip Prompt Support (v2.16 Spec Section 2.4)
**Priority: P0**

The v2.16 spec adds comprehensive Tip Prompt documentation. Currently **NOT implemented**.

**Missing in Types:**
```typescript
// SaleRequest needs:
tipAmount?: string;  // Pre-set tip amount

// Configuration needs:
configuration?: {
  enableTipPrompt?: boolean;      // Enable PIN Pad tip
  tipPromptOptions?: string[];    // ["15", "18", "20", "none"] or ["5.00", "10.00", "other"]
  // ... existing fields
}

// SaleResponse needs:
subTotalAmount?: string;  // Original amount before tip
tipAmount?: string;       // Selected tip amount
```

**Missing in UI:**
- Tip configuration section in Sale form
- PIN Pad tip options builder
- Response display showing subTotal + tip + total breakdown

**Spec Reference:**
- Section 2.4: Tip Prompt 流程（小费提示）
- Two modes: Pre-set Tip (商户指定) and PIN Pad tip (客户选择)
- Percentage format: `["15", "18", "20", "none"]` (NO % symbol)
- Fixed amount: `["5.00", "10.00", "other"]`

### 2. 🔴 Cancel Endpoint (v2.16 Spec Section 11)
**Priority: P0**

**Missing entirely:**
```typescript
// lib/payrix/types.ts
interface CancelRequest {

// lib/payrix/client.ts
async cancel(laneId: string): Promise<RequestResult<CancelResponse>>

// actions/payrix.ts
export async function cancelAction(input: BaseActionInput & { laneId: string })

// app/reversals/cancel/page.tsx
// New page needed
```

**Spec:**
- Endpoint: `POST /api/v1/cancel`
- Body: `{ laneId: number }`
- Purpose: Cancel ongoing transaction
