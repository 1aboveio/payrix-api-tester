### Codex Review Round 1 (PR #61)
- P1: E2E steps were blocking (run.sh blocks 600s, monitor kills at 180s) → made non-blocking with tmux
- P2: Timeout retry PASSED routed to E2E_SETUP (redundant) → route to DONE directly

### Codex Review Round 2 (PR #61)
- P1: `source e2e/common.sh` overrides cicd-pipeline's STATE_FILE/helpers → inline Browserless port detection instead
- P1: e2e_running.sh relied on e2e-state.json having parsed results, but raw tmux command doesn't run parser → parse TEST_OUTPUT directly for timeouts/failures/IAP errors
- P2: TIMEOUT result in retry step escalated immediately → check retry budget (maxTimeoutRetries) first

### Pattern: Non-blocking E2E in cicd-pipeline
- First tick: run setup (blocking, fast), launch tests in tmux, sleep 10s for crash detection, return
- Subsequent ticks: check tmux alive → wait. Dead → parse output, advance state
- Same pattern for e2e_running, e2e_timeout_retry, e2e_verifying
- e2e_setup stays blocking (fast, <30s)
- Never source e2e common.sh from cicd-pipeline wrappers (state file conflict)
